# Documentación Técnica — Tarjeta Personal de Dylan Ramirez

## 1. Descripción general

Este archivo HTML representa una **tarjeta de presentación personal simple**.
La página muestra información básica mi incluyendo mi nombre, una frase personal, una breve descripción sobre mi,mis pasatiempos y un medio de contacto.

Este documento está estructurado utilizando **etiquetas semánticas de HTML5**, lo que mejora la organización del contenido y facilita que navegadores, motores de búsqueda y tecnologías de accesibilidad comprendan la estructura de la página.



# 2. Decisiones semánticas

A continuación explico por qué elegí cada etiqueta utilizada en el documento.


### `<title>`

Define el **título de la página**, el cual aparece en la pestaña del navegador.



## `<body>`

Contiene **todo el contenido visible de la página**.

Dentro del body se encuentran las secciones principales de la tarjeta personal.



## `<header>`

Se utiliza para el **encabezado principal de la página**.

En este caso contiene:

* Mi nombre.
* Una frase personal.

Esto representa la **introducción de la página**.



## `<h1>`

Representa el **título principal del documento**.

Se usa para mostrar mi nombre completo:
**Dylan Ramirez Vanegas**.



## `<p>`

Se utiliza para escribir **párrafos de texto**.

En el header se usa para mostrar una frase personal.



## `<em>`

Esta etiqueta se utiliza para **dar énfasis al texto**.

La frase:
"A veces me siento mal, acomodo la silla y me siento bien"

se muestra en cursiva para indicar que es una frase destacada o personal.



## `<main>`

Contiene el **contenido principal de la página**.

Dentro de esta etiqueta se encuentran las secciones que me describen.



## `<section>`

Se utiliza para dividir el contenido en **bloques temáticos**.

En este documento hay dos secciones:

### 1. Sobre mí

Describe brevemente información personal.

### 2. Habilidades y pasatiempos

Describe intereses y actividades que disfruto.



## `<h2>`

Representa **subtítulos dentro del documento**.

Se utilizan para nombrar cada sección.



## `<time>`

Se usa para representar una **fecha de forma semántica**.

En el código se utiliza para mostrar la fecha:

13 de marzo de 2026

El atributo `datetime` permite que las máquinas interpreten la fecha correctamente.



## `<strong>`

Se utiliza para **resaltar una palabra importante** dentro del texto.

En este caso se usa para destacar la palabra **"bien"**, enfatizando la importancia de hacer las cosas correctamente.



## `<footer>`

Representa la **parte final de la página**.

Se utiliza para incluir información adicional como el contacto.



## `<address>`

Se usa específicamente para mostrar **información de contacto**.

En este caso incluye:

* Correo electrónico
* Ciudad

# 4. Conclusión

Este documento utiliza una estructura simple pero correcta de **HTML semántico**, lo cual permite:

* Organizar el contenido de forma clara.
* Mejorar la accesibilidad.
* Facilitar el mantenimiento del código.

Además, el uso de etiquetas como `header`, `main`, `section` y `footer` permite comprender rápidamente la estructura lógica de la página.
